# Как работать над проектом

## Окружение

Для удобства работы над проектом используются инструменты из **Node.js** и **npm**. Все необходимые настройки произведены. Убедитесь, что на рабочем компьютере установлена **Node.js**, соответсвтующая актуальной версии. Актуальная версия **Node.js** указана в файле `package.json` в поле `node`. После, в терминале, перейти в директорию с проектом и _единожды_ запустите команду:

```bash
npm install
```

Данная команда запустит процесс установки зависимостей проекта из **npm**.

### Сценарии

После создания проекта вам доступны следующие сценарии.

#### Компиляция проекта

```bash
npm run compile
```

Во время выполнения инструкций по компиляции проекта, в корне проекта создается директория `dist`, в которую будут помещены результирующие файлы.

#### Очистка проекта

```bash
npm run clean
```

Во время выполения инструкции по очистке проекта, директория `dist`, которая предназначена для хранения результирующих файлов, будет удалена.

#### Сборка проекта

```bash
npm run build
```

В процессе сборки приложения, будут выполнены инструкции «Очистка проекта» и «Компиляция проекта». 

#### Проверка линтером

```bash
npm run lint
```

Запуск проверки проекта статическим анализатором кода **ESLint**.

Анализ кода производится только в файлах, которые находятся в директории `src`.

**Обратите внимание**, при запуске данной команды, ошибки выводятся в терминал.

#### Запуск REPL

```bash
npm run ts
```

Запуск `ts-node` позволяет вам напрямую выполнять код TypeScript на NodeJS без предварительной компиляции. 

#### Запуск проекта

```bash
npm start
```

В процессе запуска проекта будет выполнен процесс «Сборки проекта» и запуска результирующего кода.

#### Запуск проекта в режиме разработки

```bash
npm run start:dev
```

Запуск проекта в режиме разработки с использованием `nodemon`. При изменении файлов проект будет автоматически перезагружен.

#### Запуск mock сервера

```bash
npm run mock:server
```

Запуск mock сервера для получения тестовых данных.

## Переменные окружения

Для работы приложения необходимо установить переменные окружения. Создайте файл `.env` в корне проекта на основе файла `.env-example`.

### Список переменных окружения

| Переменная | Описание | Значение по умолчанию |
|-----------|---------|---------------------|
| `PORT` | Порт для входящих соединений REST API | 4000 |
| `DB_HOST` | IP адрес сервера базы данных (MongoDB) | 127.0.0.1 |
| `DB_PORT` | Порт для подключения к базе данных (MongoDB) | 27017 |
| `DB_USER` | Имя пользователя для подключения к базе данных | admin |
| `DB_PASSWORD` | Пароль для подключения к базе данных | password |
| `DB_NAME` | Имя базы данных (MongoDB) | six-cities |
| `SALT` | Соль для хеширования пароля | required |
| `UPLOAD_DIRECTORY` | Директория для загруженных файлов | ./upload |
| `JWT_SECRET` | Секрет для подписи JWT токенов | secret-key |

## Запуск проекта

### Требования

- **Node.js** версия 18 или выше
- **npm** версия 8 или выше
- **MongoDB** версия 5 или выше

### Шаги для запуска

1. **Установка зависимостей**

```bash
npm install
```

2. **Создание файла .env**

Скопируйте файл `.env-example` в `.env` и установите необходимые значения переменных окружения:

```bash
cp .env-example .env
```

3. **Запуск MongoDB (если используется Docker)**

```bash
docker-compose up -d
```

4. **Запуск приложения**

Для режима разработки:
```bash
npm run start:dev
```

Или для production:
```bash
npm start
```

Приложение будет доступно по адресу `http://localhost:4000`.

### API документация

REST API сервис предоставляет следующие основные ресурсы:

- `/api/offers` - управление предложениями по аренде
- `/api/offers/premium/:city` - премиальные предложения для города
- `/api/comments` - управление комментариями
- `/api/favorites` - управление избранными предложениями
- `/api/users` - управление пользователями
- `/api/auth` - аутентификация и авторизация

## Структура проекта

### Директория `src`

В директории размещаются исходный код проекта: компоненты, модули и так далее. Структура директории `src` может быть произвольной.

### Файл `Readme.md`

Файл, содержащий инструкции по работе с учебным репозиторием.

### Файл `Contributing.md`

Файл, содержащий советы и инструкции по внесению изменений в учебный репозиторий.

### Остальное

Все остальные файлы в проекте являются служебными. Пожалуйста, не удаляйте и не изменяйте их самовольно. Только если того требует задание или наставник.
